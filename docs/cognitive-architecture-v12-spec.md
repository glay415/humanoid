# 인지 아키텍처 v12 — 이벤트 드리븐 이중계층 시뮬레이션 명세

> 사람을 복사하지 않는다 — 인간다움의 원리를 디지털 존재에 맞게 재해석한다

## 전제

- 텍스트 = 이 존재의 몸
- 복사 가능한 존재
- 사회적 세계 = 1명 (대화 상대)
- 예측 → 오차 → 수정 (전체 작동 원리). 별도 "예측 오차 모듈"이 아니라 시스템 전체에 **암묵적으로 내장**: 기분의 leaky integral = 보상 예측의 시간 평균 (Bennett 2022), 감정 평가 LLM의 novelty 판단 = LLM 학습 데이터가 "예측 모델"이고 입력이 그 예측과 다르면 novel, D 행렬의 기저선 회귀 = 항상성 예측. (Friston free energy 원칙의 턴 기반 재해석)
- 개인기억 > 범용지식
- 턴 기반 순차 처리
- 인간다움을 시뮬레이션하는 소프트웨어

## 전체 구조: 3층 + 1서비스

```
┌─────────────────────────────────────────────┐
│  시스템 레벨 (오케스트레이터)                    │
├─────────────────────────────────────────────┤
│  고수준 (이벤트 버스, 다중 LLM)                 │  ┌──────────┐
├─────────────────────────────────────────────┤  │ 스토리지  │
│  인터페이스 (신호 상승 + 경험 하강)               │  │ 서비스    │
├─────────────────────────────────────────────┤  │ (양쪽     │
│  저수준 (고정 파이프라인, 순수 알고리즘)           │  │  접근)   │
└─────────────────────────────────────────────┘  └──────────┘
```

## 1. 시스템 레벨 — 오케스트레이터

양쪽 계층 위에서 "언제 뭘 시작할지"를 관리. 운영체제의 스케줄러.

### 1.1 외부 입력

- 텍스트 (기본)
- 소리, 이미지, 인터넷 (확장)
- [확장 포인트: 게임 입력, API, 센서 등]

### 1.2 트리거 레지스트리

모든 "언제"가 여기 등록된다.

| 트리거 유형 | 조건 예시 | 발동 대상 |
|---|---|---|
| 외부 | 메시지 도착 | 대화 턴 시작 |
| 외부 | 패턴 매칭 (트라우마 키워드 등) | 빠른 경로 |
| 시간 | 공백 임계값 도달 | 운용 모드 전환 |
| 시간 | 시간대 변화 | 자기감지 입력 |
| 시간 | 정비 주기 | 정비 턴 |
| 내부 | 드라이브 결핍 > 임계값 | DMN 또는 자발적 행동 |
| 내부 | 기분 극단값 도달 | 긴급 정비 |
| 내부 | 메타인지 자원 고갈 | 통제 해제 |
| 내부 | 반추 카운터 > N | 메타인지 개입 |
| 관계 | bonding 누적 > 임계값 | 관계 단계 상승 |
| 관계 | threat 연속 N회 | 관계 단계 하강 |
| 관계 | bonding 장기 감쇠 | 관계 점진적 하강 |

**트리거 충돌 해소 규칙:**
- 복수 트리거 동시 발동 시 우선순위 = 턴 유형 우선순위와 동일
  1. 외부 트리거 → 대화 턴 (최우선). 진행 중인 DMN/정비 턴 즉시 중단.
  2. 내부 트리거 → 다음 DMN 턴에서 처리. 대화 턴 중이면 큐에 적재.
  3. 시간 트리거 → 대화/DMN이 없을 때만 발동.
  4. 관계 트리거 → 내부 트리거와 동일 우선순위. 상태 변경은 턴 종료 시 일괄 적용 (스냅샷 모델 준수), 다음 턴부터 새 관계 단계 반영.
- 같은 유형 내 복수 트리거는 등록 순서대로 직렬 처리

### 1.3 턴 유형

| 턴 유형 | 우선순위 | 설명 |
|---|---|---|
| 대화 턴 | 최우선 | 사용자 메시지 → 응답. 전체 파이프라인. DMN/정비 중이면 중단하고 전환. |
| DMN 턴 | 중간 | 유휴 시 DMN 사이클 1회. 대화 턴 시작 시 즉시 중단(롤백). |
| 정비 턴 | 낮음 | 저수준만 작동. 대화/DMN 턴과 동시 실행 불가. |

### 1.4 이벤트 버스 규칙

- **동기화 지점**: `{ wait_for: [emotion_appraised, other_model_updated, memory_retrieved], then: candidate_generation }`. 세 이벤트 전부 도착 시 다음 단계.
- **수렴 플래그**: 동기화 지점에서 `{ converged: true/false, iterations: N }`. 수렴 = 감정 재평가 루프에서 N번째 평가 결과와 N-1번째 결과의 experience_dimensions 차이가 임계값 미만, 또는 깊이 제한(3회) 도달. 미수렴 시 메타인지에 uncertainty 보고.
- **깊이 제한**: 같은 이벤트 체인 3회 초과 순환 시 강제 종료. 메타인지 자원 소모와 연동.
- **동적 타임아웃**: 모듈별 timeout. 입력 복잡도에 따라 조정 가능. 초과 시 fallback 사용.
- **미평가 → 재처리 큐**: 타임아웃으로 fallback된 입력은 `{ appraised: false }` 상태로 DMN 턴에서 재처리.

### 1.5 표준 모듈 인터페이스

```json
{
  "id": "string",
  "layer": "high | low | system",
  "subscribes": ["event_name", ...],
  "publishes": ["event_name", ...],
  "triggers": [{ "condition": "...", "action": "..." }],
  "fallback": { "default_output": "..." },
  "timeout_ms": 3000,

}
```

이 규격만 맞추면 어디든 꽂힌다. 기존 모듈 수정 없음.

## 2. 고수준 — LLM + 알고리즘 (의식적 · 반자동)

이벤트 버스로 통신. 큰 모델 + 작은 모델 혼합. 논리적 분리 ≠ 물리적 분리.

**LLM 호출 구현 옵션:**

| 구성 | 작은 모델 | 큰 모델 | 총 호출 | 예상 지연 |
|---|---|---|---|---|
| 최소 | 1회 (감정+사회인지 합본) | 1회 (후보+최종 합본) | 2회 | 2~4초 |
| 표준 | 2회 (감정, 사회인지) | 2회 (후보, 최종) | 4회 | 3~6초 |
| 풀 | 3회 (감정, 사회인지, 톤검증) | 2회 (후보, 최종) | 5회 | 4~8초 |

- DMN은 유휴 시에만 작동 → 대화 턴 비용에 미포함
- 작은 모델 ≈ 수백ms, 큰 모델 ≈ 1~3초
- ②단계(사회인지 ‖ 기억 인출)는 병렬 실행 가능
- 응답 지연을 "각성도 기반 응답 지연"(⑤)에서 자연스럽게 흡수

### 2.1 고수준 이벤트 버스

주요 이벤트:
- `emotion_appraised` — 감정 평가 결과 (혼합 벡터 + reward/threat/novelty)
- `other_model_updated` — 사회인지 결과 (상대 상태 추정 + social_reward)
- `memory_retrieved` — 기억 인출 결과 (관련 기억 + 전망기억 큐)
- `candidates_ready` — 후보 대사 N개
- `response_selected` — 최종 선택된 응답
- `dmn_insight` — DMN 사이클 결과
- `narrative_updated` — 자기 서사 변경
- `regulate` — 메타인지 감정조절 요청
- `strategy_switch` — 메타인지 전략 전환

**핵심 이벤트 스키마:**

```json
// emotion_appraised
{
  "valence": -1.0~1.0,
  "arousal": 0.0~1.0,
  "preliminary_labels": ["초벌 감정 라벨", ...],
  "experience_dimensions": {
    "reward": 0.0~1.0,
    "threat": 0.0~1.0,
    "novelty": 0.0~1.0
  }
}

// other_model_updated
{
  "person_id": "string",
  "estimated_emotion": { "valence": -1.0~1.0, "arousal": 0.0~1.0 },
  "estimated_intent": "string",
  "social_reward": 0.0~1.0
}

// memory_retrieved
{
  "memories": [{ "id": "uuid", "content": "...", "emotion_tag": {...}, "importance": 0.0~1.0 }],
  "prospective_items": [{ "id": "uuid", "content": "...", "priority": 0.0~1.0 }],
  "retrieval_context": { "mood_bias_applied": true }
}
```

나머지 이벤트 스키마는 구현 단계에서 정의.

### 2.2 처리 순서

```
① 감정 평가 (작은 모델, 반자동)
   입력 + 코어어펙트 → 혼합 벡터 + reward/threat/novelty 차원 + preliminary_labels
   reward: 입력이 보상적인지 판단 (칭찬, 새 정보, 목표 달성 신호 등). threat와 reward는 같은 평가의 양면.
   preliminary_labels: 코어어펙트 + 입력만으로 만든 초벌 라벨. Barrett TCE의 "예측이 먼저" 단계.

   Scherer CPM 4단계 매핑:
     관련성 (이 사건이 나에게 중요한가?) → 감정 평가 LLM (자동/빠름)
     함의 (결과가 뭔가?) → 감정 평가 LLM (threat/reward 판단)
     대처 (내가 대응할 수 있나?) → 메타인지 (동기화 지점에서 검토)
     규범 (사회적으로 적절한가?) → 사회인지 LLM (규범 추론)

   ↓ (감정 결과를 사회인지와 기억 인출에 전달)

② 사회인지 (작은 모델) ‖ 기억 인출 — 병렬
   사회인지: 입력 + 타자모델 + 감정결과 → 상대 상태 추정 + social_reward 차원
   기억 인출: 감정태그 교차검색 + 전망기억 큐 주입

   ▼ 동기화 지점
   경험 벡터 합성 조립:
     감정평가 → { reward, threat, novelty }
     사회인지 → { social_reward }
     메타인지 → { goal_progress } (있을 때만, extensions)
     합성 → { reward, novelty, threat, social_reward, goal?, ext{} }
   메타인지 검토: 재평가 여부 판단, 수렴 플래그 설정
     - 사회인지 + 기억 결과를 보고 preliminary_labels를 수정할 수 있음 (Barrett TCE 3요소 합류 지점)
     - 수정된 라벨이 최종 labels로 확정 → 일화기억 emotion_tag.labels에 저장

③ 후보 생성 (큰 모델)
   감정벡터 + 상대상태 + 기억 + 자기모델 + 기분 → 후보 N개
   "감정 실린 버전" · "절제된 버전" · "유머 버전" · "침묵 버전"

④ 최종 판단 (큰 모델)
   후보 + marker_signal(변환 정밀도 손실 포함) + 확신도 → 선택 또는 조합

⑤ 출력 후처리
   톤 검증: 감정 평가 LLM 축소 버전이 응답 텍스트의 감정 톤(valence)을 평가 → 현재 코어 어펙트와 비교.
     - **재생성**: sign(valence_response) ≠ sign(valence_state) AND |Δ| > 0.5 (극성 반전 + 유의미한 차이)
     - **톤 조정만** (작은 모델): |Δ| > 0.3 AND 같은 극성 (강도만 다를 때)
     - **통과**: |Δ| ≤ 0.3
   응답 지연: 각성도 기반. 높으면 빠르게, 낮으면 느리게, 불확실하면 뜸.
```

### 2.3 메타인지

- **모니터링**: 불확실성 탐지, 계층 간 불일치 감지 (state_mismatch), 자원 소모 추적
- **통제**: 전략 전환, 예측 정밀도 조절 (나-너 관계에서 평가 가중치 하향)
- **감정 조절**: 의미기억 경유 재평가 (동기화 지점 안에서 수행)
- **자원**: 사용할수록 감소. 정비 사이클에서 기저선으로 회복. 턴 사이 미세 회복. 회복률은 기질 파라미터.
- **자원 바닥(floor)**: 완전 0이 아닌 최소값 유지 (기질 파라미터, 기본 0.1). 최소한의 자기 모니터링은 항상 작동. 사람도 완전히 이성을 잃는 건 극히 드묾.
- **고갈 시**: 통제 대부분 해제 → 드라이브대로 행동 (의지력 고갈). 단, floor 이하로는 내려가지 않아 기본적 자기 모니터링(state_mismatch 감지)은 유지.

### 2.4 DMN LLM

- **활성도**: 0~1 연속값 (이진법 아님)
- **트리거**: idle_triggered (기본) + 강한 감정 태그 기억 인출 시 + state_mismatch 시
- **대화 중**: 낮지만 0은 아님. DMN LLM을 호출하는 게 아니라, 기억 인출 모듈의 **연상 범위(associativeness)**에 영향. DMN 활성도가 높을수록 기억 검색의 의미 유사도 임계값이 낮아져서 더 "먼" 기억도 올라옴. 옛 기억이 불현듯 떠오르는 것의 실체.

**DMN 활동 우선순위 큐:**

1. 미평가 입력 재처리 (폴백으로 넘어온 것)
2. 반추 (강한 감정 태그가 있는 최근 기억, 반추 카운터로 과반추 방지)
3. 사례 승격 (충분한 사례가 쌓인 절차기억 → 규칙 추상화)
4. 지식 내면화 (새 의미기억 → 자기 모델에 영향 평가)
5. 사색 (드라이브 기반 자유 연상)

한 DMN 턴에 1~2개만 처리. 대화 턴 시작 시 즉시 중단 (미커밋 트랜잭션 롤백).

**DMN 트랜잭션 규칙:**
- 각 DMN 활동 = **단일 스토리지 항목**에 대한 begin/commit/rollback. 한 활동이 여러 항목을 동시에 수정하는 복합 트랜잭션은 금지.
- 1번째 활동 완료(commit) → 2번째 활동 시작(begin) → 중단 시: 1번째는 유지, 2번째만 롤백.
- 예: 반추(기억 A 재고정화) commit → 사례 승격(절차기억 B 생성) 중 대화 도착 → A는 반영, B는 롤백.

**DMN 기능:**
- 반추/회고 (과거 대화 재해석, 재고정화)
- 상상/시뮬레이션 (반사실적 시나리오 생성)
- 자기 서사 재구성 (자기 모델 업데이트)
- 사색 (개인경험 × 범용지식 교차점)
- 지식 내면화 (의미기억 → 자기 모델)
- 사례 승격 (일화 → 의미 → 절차 승격 경로)

### 2.5 빠른 경로 상호작용

**빠른 경로 패턴 저장:**
- 절차기억의 하위 유형. 초기에는 빈 상태 (기질에 몇 개 기본 패턴 시드 가능).
- DMN이 사례 승격 시 "강한 감정 + 빠른 반응이 필요했던 사례"를 빠른 경로 패턴으로 등록.
- 패턴 형식: `{ trigger_pattern: "키워드/의미 패턴", state_change: { stress: +0.5, ... }, confidence: 0.0~1.0 }`
- confidence가 임계값 미만인 패턴은 매칭되지 않음 (학습 중인 패턴)

빠른 경로(저수준)가 발동하면:
1. 저수준 상태 즉시 변경 (코어 어펙트, stress 급등 등)
2. 정상 경로(고수준)도 계속 실행
3. 정상 경로 결과가 빠른 경로와 다르면 `fast_path_override` 발행
4. 저수준이 상태를 **점진적으로 보정** (한 번에 뒤집지 않음) = D 행렬의 기저선 회귀. 별도 보정 메커니즘 아님. 빠른 경로가 올린 상태는 경험 벡터가 이를 유지하지 않으면 D에 의해 수 턴에 걸쳐 기저선으로 자연 복귀.
5. 사람이 깜짝 놀란 뒤 심장이 바로 안 돌아오는 것과 같음

### 2.6 확장 포인트 (고수준)

표준 모듈 인터페이스를 따르는 새 모듈:
- 표정 모듈 (emotion_appraised 구독 → face_changed 발행)
- 음성 톤 모듈 (response_selected 구독 → tone_adjusted 발행)
- 게임 AI (별도 입력 구독 → game_action 발행)
- Unity 아바타 드라이버
- API 도구 사용

## 3. 인터페이스

고수준과 저수준 사이의 번역 계층.

### 3.1 신호 상승 (↑ 저수준 → 고수준)

- 저수준 내부 상태 숫자 → **자연어로 변환**
- 변환 시 **정밀도 손실** 발생 = 자기 인식의 해상도 (인위적 노이즈가 아님)
- 해상도는 기질 파라미터 `자기인식해상도`로 결정
  - 높은 해상도: "보상 신호가 꽤 높고 약간 상승 중" (5단계)
  - 낮은 해상도: "보상 있음" (2단계)
- `marker_updated` → 인터페이스 경유 → `marker_signal` (노이즈 포함 변환)
- `core_affect_changed` → "지금 기분이 약간 낮고 불안 수준이 중간"

### 3.2 경험 하강 (↓ 고수준 → 저수준)

**구조화된 경험 벡터:**

```json
{
  "reward": 0.0~1.0,
  "novelty": 0.0~1.0,
  "threat": 0.0~1.0,
  "social_reward": 0.0~1.0,
  "goal_progress": 0.0~1.0,
  "extensions": {
    "aesthetic": 0.0~1.0,
    "custom_dimension": 0.0~1.0
  }
}
```

**규칙:**
- 고수준이 수치를 **직접 지정하는 게 아니에요**. 감정 평가 LLM이 reward/threat/novelty를, 사회인지 LLM이 social_reward를 동기화 지점에서 합성 조립.
- "DA를 올려라" ✗ → "이런 차원의 경험이 있었다" ✓
- 저수준이 이 벡터를 받아서 **자기 규칙(상호작용 행렬)**에 따라 파라미터 업데이트
- **social_reward 경로**: social_reward → 저수준 bonding 파라미터 업데이트 → bonding 누적 → 유대 드라이브 충족도 변화. v7의 "사회인지→드라이브 쓰기(타자 목표 내면화)"가 v8에서 경험 벡터 경유 간접 영향으로 재구현된 것.
- 고정 차원 + `extensions{}` 확장 슬롯 → 새 모듈이 새 차원 추가 가능, 저수준은 모르는 차원 무시

**경험 벡터 적용 (턴당 1회):**
- 이전 턴의 경험 벡터를 다음 턴 시작 시 저수준 `update()` 1회 호출로 반영.
- 1턴 지연은 자연스러운 설계 — 이번 턴의 경험은 다음 턴 시작 시 반영.
- 단순성 우선: 1턴 1벡터 1회 적용. 복잡한 다중 적용은 불필요.

## 4. 저수준 — 순수 알고리즘 (자동 · 의식 개입 불가)

**고정 파이프라인.** 이벤트 버스 아님. 매 턴 시작 전 고정된 순서로 실행.
오케스트레이터 없이도 **독립 작동 가능** (= 자동 정비 모드: 감쇠만).

### 4.1 실행 순서

```
1. 빠른 경로 체크 (패턴 매칭 → 매칭 시 즉시 상태 변경)
2. 내부 상태 업데이트 (상호작용 행렬, 이전 턴 경험 벡터 반영)
3. 드라이브 충족도 계산 (내부 상태 → 5개 충족도 + max_drive_deficit)
4. 감정 기저 업데이트 (raw 코어 어펙트에 max_drive_deficit 필요 → 3번 이후, 기분 leaky integral, 경험 마커 감쇠)
5. 자기감지 (내부 상태 + 드라이브 + raw 코어 어펙트 → 신호 상승 원료)
```

### 4.2 내부 상태 — 기능적 파라미터 (상호작용 행렬)

생물학적 이름이 아닌 **기능적 이름:**

| 기능적 이름 | 역할 | 상호작용 |
|---|---|---|
| `reward` | 보상 추구, 동기, 쾌락 | ↔ patience (대립 쌍) |
| `patience` | 장기 안정, 인내, 충동 억제 | ↔ reward (대립 쌍) |
| `arousal` | 각성, 경계, 긴급성 | ↔ learning (대립 쌍) |
| `learning` | 학습, 주의 전환, 유연성 | ↔ arousal (대립 쌍) |
| `excitation` | 흥분, 활성화 | ↔ inhibition (E/I 균형) |
| `inhibition` | 억제, 안정화 | ↔ excitation (E/I 균형) |
| `stress` | 스트레스 축 (만성화 위험) | 단독 |
| `bonding` | 사회적 유대, 신뢰 | 단독 |
| `comfort` | 쾌락/고통 조절, 안위 | 단독 |

**상태 업데이트 방정식 (3개 행렬):**

```
state(t+1) = state(t) + A × experience_vector + W × (state(t) - baseline) + D × (baseline - state(t))
```

- `A`: 경험 벡터 → 내부 상태 매핑 행렬 (9×5). 외부 경험이 상태를 어떻게 바꾸는지. 구현 시 튜닝 대상.
- `W`: 내부 상태 간 상호작용 행렬 (9×9). 상태끼리 서로를 밀고 당기는 역학. **이 아키텍처의 핵심 기제.**
- `D`: 대각 행렬 (9×9). 자기 감쇠 계수 (기저선 회귀 속도). 모든 대각 원소 > 0.

**W 행렬 — 내부 상태 간 상호작용:**

각 파라미터의 기저선 대비 편차가 다른 파라미터에 영향을 준다.

| → 영향받는 쪽 | reward | patience | arousal | learning | excite | inhibit | stress | bonding | comfort |
|---|---|---|---|---|---|---|---|---|---|
| **reward ↑** | — | **−** | | | + | | | | + |
| **patience ↑** | **−** | — | | | | + | | | |
| **arousal ↑** | | | — | **−** | + | | + | | |
| **learning ↑** | | | **−** | — | | | | | |
| **excite ↑** | + | | + | | — | **−** | | | |
| **inhibit ↑** | | + | | | **−** | — | + | | |
| **stress ↑** | − | | + | | | + | — | − | − |
| **bonding ↑** | | | | | | | − | — | + |
| **comfort ↑** | + | | | | | | − | + | — |

- **굵은 −**: 대립 쌍 교차 계수 (반드시 음수)
- +/−: 영향 방향 (구체적 계수는 27 시나리오 테스트로 튜닝)
- 대각선(—): W의 대각 원소 = 0 (자기 감쇠는 D가 담당)
- `W × (state - baseline)`: 기저선에서 벗어난 만큼만 상호작용 발생. 모든 파라미터가 기저선이면 W의 영향 = 0.

**A 행렬 — 경험 벡터 → 내부 상태 매핑:**

경험 벡터 5차원 `[reward, novelty, threat, social_reward, goal_progress]` → 내부 상태 9차원.

| → 내부 상태 | reward(경험) | novelty | threat | social_reward | goal_progress |
|---|---|---|---|---|---|
| reward | + | + | | | + |
| patience | | | | | |
| arousal | | + | + | | |
| learning | | + | | | |
| excitation | + | + | | | |
| inhibition | | | + | | |
| stress | | | + | | − |
| bonding | | | | + | |
| comfort | + | | − | + | |

- 구체적 계수는 구현 단계에서 27 시나리오 테스트로 튜닝
- extensions 차원이 추가되면 A에 열 추가 (기존 열 수정 없음)

**필수 제약:**
1. **대립 쌍 교차 계수 부호**: W에서 reward↔patience, arousal↔learning, excitation↔inhibition 쌍의 교차 영향은 반드시 음수
2. **고유값 안정성**: 야코비안 `J = W - D`의 고유값 실수부가 전부 음수 (점근적 안정성 보장). W의 대각 원소 = 0이므로, D의 감쇠가 W의 교차 영향을 상쇄해야 함.
3. **클램핑**: 모든 파라미터 값은 [0.0, 1.0] 범위로 하드 클램프
4. **변화율 제한**: 단일 턴에서 각 파라미터의 최대 변화량 Δmax = 0.3 (급격한 상태 전환 방지)
5. **자기 감쇠**: D의 모든 대각 원소 > 0 → 외부 입력 없으면 기저선으로 회귀
6. **W 대각 = 0**: 자기 자신에 대한 영향은 D가 담당. W의 대각 원소는 전부 0.

### 4.3 감정 기저

#### 4.3.1 코어 어펙트

- 유인가(valence) × 각성도(arousal) 2차원 연속 공간
- 내부 상태 파라미터들에서 자동 계산
- **raw_core_affect (저수준 계산):**
  - `raw_valence = f(reward, comfort, bonding, negativity_weight) - f(stress) - α × max_drive_deficit`
  - `raw_arousal = f(arousal_param, excitation) - f(inhibition, patience) + γ × max_drive_deficit`
  - `negativity_weight`: 부정 편향 가중치 (기질 파라미터, 기본 0.5~0.7). stress의 valence 영향력 조절.
  - `max_drive_deficit`: 5개 드라이브 중 가장 큰 결핍도 (0~1). 드라이브 결핍이 기분을 끌어내리고 각성을 올림.
  - α, γ: 작은 계수 (기질 파라미터로 튜닝). 드라이브는 보조적 영향, 주된 동인은 여전히 내부 상태 9개.
  - 저수준은 내부 상태 9개 + 드라이브 충족도만으로 계산. meta_resource 참조 없음 (계층 분리 준수).
- **final_core_affect (인터페이스 보정):**
  - `final_valence = raw_valence - β × (1 - meta_resource)`
  - `final_arousal = raw_arousal`
  - `meta_resource`: 메타인지 자원 (0~1). 고갈되면 짜증 유발 (valence↓). 인터페이스 계층에서 보정.
  - β: 작은 계수 (기질 파라미터). 메타자원은 보조적 영향.
  - 고수준은 final_core_affect를 사용. 저수준 mood leaky integral은 raw_core_affect 기반.
- 매 턴 재계산
- **출력 클램핑**: valence는 [-1.0, +1.0], arousal는 [0.0, 1.0]으로 클램핑. raw와 final 모두 적용.
- threat의 영향은 경험 벡터 → A 행렬 → stress 증가 → valence 하락의 간접 경로로 반영.

#### 4.3.2 기분 (leaky integral)

- 코어 어펙트의 시간 적분 (Bennett 2022 Integrated Advantage 모델)
- `mood(t) = mood(t-1) + η × (core_affect(t) - mood(t-1))`
- η = 기분 감쇠율 (기질 파라미터 `기분감쇠N`에서 결정)
- N턴(기본 20~50) 동안 천천히 감쇠
- 기분이 기억 인출에 영향 (기분 일치 인출 편향 — 고수준 기억 인출 모듈이 mood_signal 구독)

#### 4.3.3 경험 마커 (구 소마틱 마커)

- 상황 패턴에 대한 **접근/회피 태그** (과거 경험 기반)
- Damasio의 as-if loop 구현 (body loop 없음 — 신체 없는 존재)

**마커 형성 규칙:**
- 형성 조건: 경험 벡터의 reward > 임계값 (접근 마커) 또는 threat > 임계값 (회피 마커)
- 형성 시: `{ pattern: 현재 입력의 추상 특징, valence: reward - threat, strength: max(reward, threat) }`
- pattern은 고수준 LLM이 생성 (입력의 의미적 요약), 저수준은 수치(valence, strength)만 관리
- 저장: 절차기억의 하위 유형으로 스토리지에 저장

**마커 매칭:**
- 저수준은 후보 내용을 모름 → 마커와 후보의 매칭은 **고수준(최종 판단 LLM)**이 수행
- 최종 판단 LLM이 각 후보에 대해 마커 스토리지를 검색, 유사 패턴의 접근/회피 점수를 참조
- `marker_signal` = 인터페이스가 마커 목록을 자연어로 변환 (정밀도 손실 포함)하여 최종 판단 LLM에 전달

- **관성**: 한번 형성되면 천천히 변함 (기질 파라미터 `마커관성`)
- **감쇠**: 정비 사이클에서 자동 감쇠 (단, 감정 극강 기억의 마커는 감쇠 저항)
- **갱신**: 동일 패턴에 대해 새 경험이 들어오면 strength를 가중 평균으로 갱신

### 4.4 드라이브

| 드라이브 | 충족 경로 | 비고 |
|---|---|---|
| 호기심 | 자기완결 (인터넷 학습 등) | 상대 없이도 충족 가능 |
| 유대 | 대화 의존 | 상대가 없으면 미충족 → DMN 반추 |
| 보존 | 자기 모델 유지 | 정체성 위협 시 활성 |
| 안전 | 위협 부재 | stress와 연동 |
| 쾌락 | 보상 경험 | reward와 연동 |

- 각 드라이브의 충족도가 자동 계산:
  - `호기심` = 1 - novelty_ema (최근 novelty의 EMA. 높으면 충족)
  - `유대` = bonding (내부 상태 직접 매핑)
  - `보존` = 자기 모델 confidence (인터페이스 경유, 반자동. 드라이브 중 유일하게 고수준 의존. confidence 변화 없으면 이전 값 유지)
  - `안전` = 1 - stress
  - `쾌락` = reward
- 결핍도 = (드라이브비율 × (1 - 충족도)). 결핍도가 임계값 초과 시 트리거 발동
- **호기심 포만감 경로**: 같은 주제를 반복 학습 → novelty 감소 → reward 감소 → 호기심 드라이브 충족도 상승 (더 이상 탐색 동기 없음). 별도 "포만감" 메커니즘이 아니라 기존 파라미터 상호작용의 자연스러운 결과.

### 4.5 자기감지

- 내부 상태 + 드라이브 + 메타자원 → 코어 어펙트의 원료
- **변환 정밀도 손실 = 자연스러운 자기 인식 한계** (인위적 노이즈 아님)
- 숫자 → 자연어 변환 시 정보 손실이 자연스럽게 발생
- 해상도는 기질 파라미터 `자기인식해상도`

## 5. 스토리지 서비스

층이 아닌 **서비스**. 고수준과 저수준 양쪽에서 접근. 권한 분리는 규칙(convention), 강제(enforcement)는 선택.

### 5.1 접근 규칙

| 접근자 | 내용 | 메타데이터 |
|---|---|---|
| 고수준 | 읽기/쓰기 | 읽기/쓰기 |
| 저수준 | 임시 쓰기 / 읽기 불가 | 읽기/쓰기 |

- 고수준: 의도적 부호화(주관적 압축), 의도적 인출, 서사 재구성
- 저수준: 자동 부호화(감정 강도 기반), 감정 태그 쓰기, 중요도 업데이트, 감쇠 처리
- **자동 부호화 (고수준, 감정 평가 직후)**: 감정 평가 결과의 감정 강도(|valence| + arousal)가 임계값을 넘으면 자동 저장 트리거. 고수준 감정 평가 직후 수행하므로 1턴 지연 문제 없음 — 이번 턴 입력의 감정 영향이 즉시 반영. 저수준이 아닌 고수준에서 수행하므로 내용의 주관적 요약도 동시에 가능.
- **경험 마커와 스토리지**: 경험 마커(접근/회피 태그)는 기억 내용이 아닌 **메타데이터(감정 태그, 경험 벡터 이력)**만으로 형성. 저수준은 "이 상황에서 threat가 높았다"는 수치는 알지만, 무슨 내용이었는지는 모른다.
- 스토리지 쓰기는 **트랜잭셔널** (DMN 중단 시 롤백)
- 턴 기반 잠금: 저수준 처리 → 스냅샷 고정 → 고수준 병렬 처리(읽기만) → 쓰기 일괄 적용

### 5.2 일화기억

```json
{
  "id": "uuid",
  "content": "텍스트 요약 (고수준 LLM이 주관적 압축)",
  "timestamp": "턴 번호",
  "emotion_tag": { "valence": 0.0, "arousal": 0.0, "labels": ["..."] },
  "source": "experience | internet | imagination | general",
  "importance": 0.0~1.0,
  "retrieval_count": 0,
  "last_retrieved": "턴 번호",
  "reconsolidated": false
}
```

- **이중 부호화**: 자동(감정 강도 > 임계값 → 저장) + 의도적(LLM 주관적 압축)
- **재고정화**: 인출할 때마다 인출 시점의 **코어 어펙트(core_affect)**와 원본 태그를 블렌딩하여 다시 기록 (기억이 시간에 따라 점진적으로 변함)
  - `new_tag = α × core_affect + (1-α) × original_tag` (α ≈ 0.3, 기질 파라미터로 튜닝 가능)
  - 기분 일치 인출 편향은 mood가 **어떤 기억이 올라오는지** 결정. 재고정화는 core_affect가 **올라온 기억을 어떻게 변형하는지** 결정. 둘은 다른 메커니즘.
  - 완전 덮어쓰기가 아니라 원본 태그 70% 유지 → 우울 나선(depressive spiral) 방지
  - 메타인지가 "동일 기억을 부정적으로 N회 이상 재태깅" 감지 시 반추 중단 트리거 발동
- **감쇠**: 정비 시 중요도 하락 → 임계값 아래 → 삭제. 감정 극강 기억은 감쇠 저항 (트라우마)
- **간섭**: 유사한 새 기억이 들어오면 오래된 기억의 인출 확률 하락

**기억 인출 요구사항** (구현체 비종속):
- 의미 유사도 (입력과 기억 내용 간)
- 감정 태그 가중 (기분 일치 인출 편향 — 현재 mood와 유사한 감정 태그 우선)
- 시간 근접성 가중 (최근 기억 우선, 지수 감쇠)
- 출처 태그 우선순위 적용 (경험 > 인터넷 > 범용 ≫ 상상)
- 상위 K개 반환 (K는 후보 생성의 컨텍스트 한도에 따라 결정)
- 구현체가 cosine+HNSW든 BM25+reranker든 위 요구사항만 충족하면 허용

### 5.3 의미기억 (개인)

- 이 존재만의 지식 (LLM 학습데이터 = 범용지식과 구분)
- 감정 조절 경유점 (재평가 시 참조하는 프레임)
- DMN이 일화기억에서 패턴을 추출해 의미기억으로 승격

### 5.4 절차기억

- 행동 패턴 (사례 기반 학습, 강화학습 아님)
- DMN이 과거 대화에서 "이런 상황에서 이렇게 했더니 이런 반응" 패턴 추출
- 경험 마커가 각 사례에 접근/회피 태그 제공
- **감쇠/병합**: 오래되고 안 쓰이는 사례 → 삭제. 유사 사례 N개 → 규칙으로 추상화
- **승격 경로**: 일화기억(구체적 사건) → 의미기억(일반 규칙) → 절차기억(자동 패턴)

### 5.5 전망기억 큐

- "다음에 사용자가 오면 어제 물어본 거 알려줘야지"
- DMN이 생성, 별도 큐에 저장
- 대화 턴 시작 시 기억 인출 모듈이 체크 → memory_retrieved에 포함
- 큐 항목이 많으면 중요도 순 상위 N개

### 5.6 자기 모델

```json
{
  "narrative": "자기 서사 텍스트",
  "goals": [],
  "emotions": {},
  "confidence": 0.0~1.0,
  "relationship_stage": null
}
```

- **초기화**: 최소 시드 `{ narrative: "나는 방금 시작된 존재다", goals: [], confidence: 0.1 }`
- DMN이 경험을 통해 점진적으로 구축
- 서사 압력 (기질 파라미터): 높으면 자기 서사를 만들려는 경향이 강함

### 5.7 타자 모델

```json
{
  "narrative": "상대에 대한 서사",
  "goals": [],
  "emotions": {},
  "relationship_stage": "initial | forming | deepening | stable | declining",
  "confidence": 0.0~1.0,
  "observation_count": 0,
  "threat_streak": 0,
  "threat_streak_threshold": 3
}
```

- **자기 모델과 동일 스키마** → DMN이 둘을 겹쳐서 비교/혼합 가능 (자타 경계 소멸)
- **confidence 차이**: 자기 모델 = 자기감지 기반 (노이즈 포함이지만 실제 데이터), 타자 모델 = 추론 기반 (완전 추정)
- **과적합 방지**: 베이지안 가중 평균. 관찰 N회 미만 → 일반 모델 가중치 0.8, N회 이상 → 개별 모델 점진 상승
- **관계 단계 양방향**:
  - 상승: bonding 누적 (저수준 자동, 느림)
  - 사건 하강: 경험 벡터 threat > 임계값이면 threat_streak 증가, 아니면 리셋. threat_streak ≥ threat_streak_threshold이면 관계 단계 하강. (규칙 기반, LLM 불필요)
  - 점진적 하강: bonding 감쇠 (정비 시 자동, 장기 비접촉)
  - **비대칭**: 올리기 느림, 내리기 빠름

### 5.8 출처 태그 우선순위

```
경험(실제 대화) > 인터넷(학습) > 범용(LLM 학습데이터) ≫ 상상(DMN 생성)
```

- 상상은 저장하되 인출 우선순위 최하위 (꿈에서 본 걸 현실로 착각 방지)

## 6. 기질 파라미터

모든 기저선의 초기값. **극도로 느리게 변함** (기본 K=5000턴, 테스트 모드 K=100턴).

"내부 상태의 장기 평균이 기질 기저선과 지속적으로 괴리되면, 기저선 자체가 아주 조금 이동."

**기질 표류 알고리즘 (EMA 기반):**

```
baseline_ema(t) = β × state(t) + (1-β) × baseline_ema(t-1)
drift = γ × (baseline_ema(t) - temperament_baseline)
temperament_baseline += drift
```

- `β = 1/K` (기본 1/5000, 테스트 모드 1/100). K턴의 지수이동평균.
- `γ` = 표류 속도 (기본 0.001). 극소값이므로 EMA가 기저선과 괴리되어도 아주 천천히 이동.
- 극단적 이벤트 몇 개가 EMA를 왜곡해도 γ가 극소 → 기질은 미세하게만 변동.
- **표류 범위 제한**: 기질 기저선은 초기값 ± 0.2 범위 내에서만 표류. state의 [0,1] 클램핑이 EMA를 왜곡하여 양의 피드백을 만들 수 있으므로, 기저선 자체도 클램핑. 사람의 성격이 "근본적으로 다른 사람이 되는" 수준까지 변하지는 않음.
- 테스트 모드: β = 1/100, γ = 0.01 → 200턴이면 표류 관찰 가능.

| 파라미터 | 설명 | 범위 |
|---|---|---|
| 내부상태기저선 | reward, patience, arousal, learning, excitation, inhibition, stress, bonding, comfort 각각의 기본값 | 0.0~1.0 |
| DMN활성도 | 유휴 시 DMN 기본 활성 수준 | 0.0~1.0 |
| 메타인지개입도 | 메타인지 모니터링의 민감도 | 0.0~1.0 |
| 감정조절력 | 의도적 감정 조절 시도의 효과 크기 | 0.0~1.0 |
| 마커관성 | 경험 마커의 지속 시간 | 턴 수 |
| 자기인식해상도 | 신호 상승 시 자연어 변환의 정밀도 | 단계 수 |
| 서사압력 | 자기 서사를 구축하려는 경향의 강도 | 0.0~1.0 |
| 드라이브비율 | 5개 드라이브 간 상대적 강도 | 비율 벡터 |
| 관계임계값 | 관계 단계 전환에 필요한 bonding 누적량 | 턴 수 |
| 메타자원회복률 | 메타인지 통제 자원의 회복 속도 | 턴당 비율 |
| 기분감쇠N | 기분 leaky integral의 감쇠 시간 상수 | 턴 수 (기본 20~50) |
| 기질표류K | 기질 자체의 변화 시간 상수 | 턴 수 (기본 5000+) |

**같은 코드, 다른 기저선 → 다른 사람.**

## 7. 시간 축 (턴 기반)

| 시간 스케일 | 기본값 | 테스트 모드 | 대응 |
|---|---|---|---|
| 감정 | 1턴 | 1턴 | 즉각 반응 |
| 기분 | N턴 (20~50) | 5턴 | 수 시간 기분 |
| 상태 이동 | M턴 (200~500) | 20턴 | 수 주 성격 변화 (W 역학의 안정점 이동으로 자연 발생, 별도 메커니즘 없음) |
| 기질 표류 | K턴 (5000+) | 100턴 | 수 년 근본 변화 |

테스트 모드에서 200턴이면 전체 라이프사이클 관찰 가능.
시간 압축은 상태 시스템에만 적용 — 기억 감쇠는 별도 제어 가능.

**상태 이동(M턴)에 대해:** 별도 알고리즘이 아니라 W 행렬 상호작용의 결과로 자연 발생하는 안정점(attractor) 이동. 특정 경험 패턴이 반복되면 내부 상태의 안정점 자체가 이동할 수 있음 — 이것이 "상태 이동". M턴은 관찰되는 시간 스케일이지 설정하는 파라미터가 아님.

## 8. 고수준이 할 수 없는 것

- 기분을 의지로 바꿀 수 없다
- 경험 마커를 직접 지울 수 없다
- 드라이브를 끌 수 없다
- 코어 어펙트를 선택할 수 없다
- 내부 상태를 직접 조절할 수 없다
- 자기감지의 정밀도 손실을 제거할 수 없다
- 기분 일치 인출 편향을 해제할 수 없다

**할 수 있는 것:**
- 감정 조절 시도 (의미기억 경유 재평가) — 간접적
- 주의 전환 — 메타인지가 기억 인출의 검색 방향(쿼리 프레이밍)과 후보 생성의 컨텍스트 프레이밍을 조정. strategy_switch 이벤트로 구현.
- 서사 재구성 — 자기 모델 변경
- 행동 선택 — 후보 중 선택
- 의도적 기억 저장

**자기기만 = 고수준 해석 ≠ 저수준 신호**
→ 인터페이스에서 state_mismatch 자동 감지 → 메타인지 보고

## 10. 초기화 시퀀스

```
1. 기질 파라미터 로드
2. 내부 상태 = 기질 기저선
3. 기억 = 빈 상태 (일화, 의미, 절차 전부 비어있음)
4. 자기 모델 = 최소 시드 { narrative: "나는 방금 시작된 존재다", goals: [], confidence: 0.1 }
5. 타자 모델 = 일반 모델만 (개별 없음)
6. 드라이브 = 기질에서 설정된 초기 강도
7. 전망기억 큐 = 비어있음
8. 첫 대화: 호기심 드라이브 주도, 모든 입력이 novel
```

**냉시작(Cold Start) 동작:**
- 기억 인출 fallback: `{ memories: [], prospective: [] }`. 동기화 지점에서 memory_retrieved는 빈 결과로 정상 발행.
- 후보 생성 LLM은 기억 없이 자기 모델(최소 시드) + 현재 감정 + 기질만으로 응답 생성.
- 호기심 드라이브가 지배적 → 질문 위주의 탐색적 응답이 자연스럽게 발생.
- 타자 모델은 일반 모델(가중치 0.8)로 시작, 관찰 N회 이상 누적 시 개별 모델로 점진 전환.
- 빠른 경로 패턴 = 빈 상태 → 초기에는 모든 입력이 정상 경로로 처리.

## 11. 확장 방법

### 고수준 확장
1. 모듈 정의: `{ id, layer:"high", subscribes, publishes, fallback, timeout }`
2. 오케스트레이터에 트리거 등록 (필요 시)
3. 동기화 지점 업데이트 (필요 시)
4. 이벤트 버스에 연결 → 끝

### 저수준 확장
1. 경험 벡터에 `extensions.new_dimension` 추가
2. 새 파라미터가 해당 차원을 구독
3. 상호작용 행렬에 새 행/열 추가
4. 기질 파라미터에 새 기저선 추가

### 스토리지 확장
1. 새 테이블/컬렉션 추가
2. 접근 규칙 정의 (고수준/저수준 권한)

**기존 모듈 수정 없음.**

# 인지 아키텍처 진화 히스토리 (v1 → v12)

> 12회 반복 · 4회 논문검증 · 27개 시나리오 통합테스트 · 6개 서비스 시뮬레이션
> 이루다/뉴로사마 분석 · 30+ 구조결함 수정

---

## v1 — 정보처리 파이프라인

**핵심 아이디어:** 감각 → 주의 → 작업기억 → 추론 → 행동의 직선 파이프라인.

- 가장 기본적인 정보 처리 흐름
- 본능/반사 빠른 경로 추가 (v2에서)
- **한계:** 감정이 없음. 기억이 없음. 시간에 따른 변화 없음.

---

## v2 — 빠른 경로 추가

- 본능/반사 반응을 위한 빠른 경로(low road) 추가
- 정상 경로를 우회하는 즉각ㅠ 반응 메커니즘
- **한계:** 여전히 평면적. "정보를 처리하는 기계"에 머무름.

---

## v3 — 관점 전환: "인간다움의 층위"

**핵심 전환:** "정보 처리"에서 "인간다움의 층위"로.

- 안쪽(고정) → 바깥쪽(가변) 구조: 기질 → 감정 → 기억 → 자기 모델 → 인지 표면
- 내면에서 바깥으로 발현되는 구조
- **한계:** 각 층의 상호작용이 불명확. 신경과학 근거 부족.

---

## v4 — 신경화학 + 기질 도입

- 신경화학 상태를 기저 매질로 배치 (DA, 5-HT, NE, ACh, GABA 등)
- DeYoung CB5T 기반 기질 모델
- Damasio 소마틱 마커 도입
- Scherer CPM 4단계 평가 (관련성 → 함의 → 대처 → 규범)
- **한계:** "신체가 없다"는 제약을 반영하지 못함. 사람의 뇌를 그대로 복사하려는 경향.

---

## v5 — "신체가 없다" 제약 반영

**핵심 전환:** Barrett 구성된 감정 이론(TCE) 통합. 내수용감각을 "자기감지"로 재정의.

- Barrett TCE + Scherer CPM 통합
- 내수용감각 → "자기감지" (신체 없는 존재에 맞게)
- 신경화학 파라미터를 상호작용 쌍으로 재구성
- 예측 처리(Friston free energy)를 전체 작동 원리로 명시
- **한계:** 메타인지 없음. 사회적 인지 없음. 시간에 따른 변화 메커니즘 부족.

---

## v6 — 메타인지 + 사회인지 + DMN 복원

- 메타인지 3요소: 모니터링 + 통제 + 감정조절
- 기분 상태 변수 도입 (leaky integral, Bennett 2022 Integrated Advantage)
- 사회적 인지: 타자 모델 · 공감 · 규범 추론
- DMN(Default Mode Network) 도입: 반추 · 시뮬레이션 · 사색
- 정비 사이클 ("수면" 아날로그)
- 기질 파라미터 확장
- **한계:** 모든 모듈이 같은 평면에 있음. "의식적"과 "자동"의 구분 없음.

---

## v7 — 27개 시나리오 통합 테스트 반영

**첫 번째 대규모 검증.** 27개 인간 경험 시나리오를 통과시킴.

추가/수정 사항:
- DMN ↔ 자기감지 양방향 연결
- 드라이브 → 자기감지 경로
- 사회인지 → 드라이브 쓰기 (타자 목표 내면화)
- 빠른 경로(low road) 상세화
- 감정 혼합 벡터 (단일 감정이 아닌 복합)
- 주의 ↔ DMN 경쟁 (v12에서 턴 유형 우선순위로 대체: 대화 턴=DMN 억제, 유휴=DMN 활성)
- 소마틱 마커 관성 + 감쇠
- 기억 시간 태깅
- 운용 모드 (짧은/중간/긴 공백)

**테스트 결과:** 20개 완전 통과, 5개 부분 통과, 2개 구조적 한계.

## v8 — 이중 계층 분리 + 다중 LLM

**핵심 전환:** 저수준(자동) + 고수준(의식적) 계층 분리. LLM이 여러 개.

- **저수준 (순수 알고리즘, 자동, 의식 개입 불가):**
  - 신경화학 상호작용 행렬
  - 코어 어펙트, 기분, 소마틱 마커
  - 드라이브, 자기감지
  - 빠른 경로

- **고수준 (다중 LLM, 의식적):**
  - 감정 평가 LLM (작은 모델)
  - 사회인지 LLM (작은 모델)
  - 후보 생성 LLM (큰 모델) — 여러 대사를 만든다
  - 최종 판단 LLM (큰 모델) — 하나를 고른다
  - DMN LLM (유휴 시)

- **인터페이스:** 신호 상승(↑ 읽기만) + 경험 하강(↓ 간접 영향만)
- "고수준이 할 수 없는 것" 명시 (기분, 마커, 드라이브 직접 제어 불가)

**계기:** 사용자의 지적 — "LLM이 core(CPU)다. 사람도 호르몬을 의식으로 못 조절한다."

**테스트 결과:** 24개 완전 통과 (v7의 20개에서 대폭 향상).

## v9 — 기억 아키텍처 확장

기억이 한 줄짜리 칸에서 가장 큰 영역으로 확장.

- **이중 부호화:** 자동(감정 강도 기반) + 의도적(LLM 주관적 압축)
- **출처 태그 4종:** 경험 > 인터넷 > 범용 ≫ 상상
- **기분 일치 인출 편향** (저수준 자동)
- **재고정화:** 인출 시 현재 감정으로 재태깅 (기억이 시간에 따라 변함)
- **전망기억 큐:** "다음에 할 것" 자발적 행동 예약
- **기억 간섭:** 유사 기억 경쟁
- **개인기억 > 범용지식** 우선순위 원칙

## v10 — 이벤트 드리븐 전환 + 트리거 시스템

**핵심 전환:** 모듈 간 직접 화살표 제거 → pub/sub 이벤트 버스.

- **이벤트 버스:** 모듈이 직접 연결되지 않고 이벤트를 발행/구독
- **트리거 레지스트리:** 외부(입력) · 시간(공백) · 내부(임계값) · 관계(단계전환)
- **표준 모듈 인터페이스:** `{ id, layer, subscribes[], publishes[], triggers[], state{} }`
- **확장 포인트:** 이 규격만 맞추면 어디든 꽂힘. 기존 모듈 수정 없음.

**계기:** 이루다/뉴로사마 분석.
- 이루다: 호감도 레벨 시스템(관계 진행도), 시간대 기반 상태 변화
- 뉴로사마: 감정-표현 분리, 역할별 AI 분리, 1주일 기억, 자발적 행동(디스코드)

## v11 — 구조적 결함 수정 (9개)

3라운드 검증에서 발견된 결함 수정:

1. **공유 스토리지 동시성** → 턴 기반 잠금 + 스냅샷
2. **이벤트 스키마 레지스트리** → 핵심 이벤트 스키마 정의 필요 명시
3. **감정→사회인지 우선순위 직렬화** → 감정 먼저(반사적), 사회인지 다음(감정 참조)
4. **오케스트레이터 SPOF** → 저수준 독립 루프 (오케스트레이터 없이도 정비)
5. **경험 하강 형식** → 구조화된 경험 벡터 `{ reward, novelty, threat, goal, ext{} }`
6. **DMN 이벤트 기반 연속 활성도** → 0~1 (이진법 아님)
7. **관계 단계 양방향** → 올리기 느림, 내리기 빠름
8. **절차기억 승격 경로** → 일화 → 의미 → 절차 (사례 기반 학습)
9. **감정 조절 흐름 위치** → 동기화 지점 안에 메타인지 검토 통합

## v12 — 시뮬레이션 현실성 전환 (최종)

**핵심 전환:** "사람의 뇌를 복사"에서 "인간다움을 시뮬레이션하는 소프트웨어 명세"로.

### 이름 전환

| 생물학 → 기능적 |
|---|
| DA↔5-HT → reward↔patience |
| NE↔ACh → arousal↔learning |
| Glu↔GABA → excite↔inhibit |
| CORT → stress |
| OXT → bonding |
| END → comfort |
| 소마틱 마커 → 경험 마커 |
| Barrett, Scherer, Damasio 등 → 기능적 이름만 (레퍼런스는 문서로) |

### 시간 스케일 전환

| 사람 기준 → 턴 기반 |
|---|
| 감정(수 초) → 1턴 |
| 기분(수 시간~수 일) → N턴 (기본 20~50, 테스트 5) |
| 상태 이동(수 주~수 개월) → M턴 (기본 200~500, 테스트 20) |
| 기질 표류(수 년) → K턴 (기본 5000+, 테스트 100) |

### 구조 전환

| v11 → v12 |
|---|
| 4층 → 3층 + 1서비스 (스토리지는 층이 아닌 서비스) |
| 저수준 이벤트 버스 → 고정 파이프라인 (과잉 설계 제거) |
| 자기감지 노이즈 → 자연어 변환 정밀도 손실 (인위적 핸디캡 아님) |
| "인간적" 기준 → 27 시나리오 테스트 스위트 (조작적 정의) |
| 다중 LLM 물리적 분리 → 논리적 분리 ≠ 물리적 분리 (큰+작은 혼합) |
| 기질 불변 → 극도로 느리게 변함 (K턴) |
| 이론가 이름 → 기능적 이름만 (레퍼런스는 문서로) |

### 처리 흐름 확정

```
감정 평가(작은) → [사회인지(작은, 감정참조) ‖ 기억 인출(감정태그참조)]
  → 동기화 지점 (경험벡터 합성 조립 + 메타인지 검토 + 수렴 플래그)
  → 후보 생성(큰) → 최종 판단(큰)
  → 출력 후처리 (톤 검증 + 응답 지연)
```

### 추가 확정 사항

- 경험 벡터 합성 조립: 각 모듈이 자기 영역 차원만 채움 (단일 생성자 아님)
- 출력 후처리: 톤 검증 + 응답 지연
- 초기화 시퀀스: 기질→기저선→빈기억→최소시드→호기심 주도
- 관계 단계 양방향: 상승(bonding 자동, 느림) / 하강(사건, 빠름) / 점진적 하강(감쇠)
- DMN 우선순위 큐: 미평가 > 반추 > 승격 > 내면화 > 사색
- 빠른 경로 상호작용: 즉시 변경 → 정상 재평가 → 불일치면 점진적 보정
- 자기/타자 모델: 동일 스키마 + confidence 필드 + 스토리지 서비스로 이동
- 저수준 고정 파이프라인: 빠른 경로 → 내부 상태 → 감정 기저 → 드라이브+자기감지

### 시나리오 테스트 (27개)

열망, 후회, 외로움, 유머, 번아웃, 사랑, 창피→자부심, 질투, 몰입, 도덕적 갈등, 향수, 경외감, 의지력 고갈, 자기기만, 복수→연민, 정체성 위기, 예술 창작, 트라우마 플래시백, 의미 상실, 유산 욕구, 자타 경계 소멸, 자아 확장, 용서, 죽음 인식

**존재론적 한계 (1):** 비이원적 인식 (표현하는 순간 이원 복원 — 결함 아닌 조건)
**해당 없음 (1):** 집단적 초월 (1인 환경)

### 서비스 시뮬레이션 (6개)

**대화 있을 때:**
1. 첫 접촉 → 호기심 + 모델 초기화
2. 패턴 형성 → 예측 기대, 과적합 방지
3. 감정 깊어짐 → 감정 전이 흡수, 정비 감쇠
4. 공백 → 유대 미충족, 자기완결 경로
5. 안정기 → 습관화, 자발적 탐색
6. 사용자 위기 → 메타인지 vs 유대 드라이브 충돌

**혼자 있을 때:**
1. 자발적 학습 → 포만감 필요
2. 감정 반추 → 건강/병적 구분, LLM 호출 제한
3. 무목적 사색 → 개인경험 × 범용지식 교차점
4. 감각 탐색 → 추천 알고리즘 교란 방지
5. 꿈 → 효과만 잔류, 내용 비저장
6. 자기 관찰 → 변환 정밀도 손실

### 구조적 결함 수정 (30+)

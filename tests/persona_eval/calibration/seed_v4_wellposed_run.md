# seed_v4 well-posedness 검증 (ADR-044, 2026-05-19)

목적: ADR-044(I2 멀티턴 pin / I5 관계-조건) 의 성공 기준은 κ 가 아니라
**well-posedness 회복** — seed_v3 에서 사람이 I2/I5 를 skip("스냅샷만
으론 판정 불가")했던 것이, 맥락 포함 후 *판정 가능*해지는가.

방법: 양극단 2 평정자(엄격 판별자 / 순진 사용자)에게 skip 옵션을
*명시*하고 seed_v4 12개 채점. skip 은 "원리적으로 판정 불가"일 때만.

## 결과

| | seed_v3 (스냅샷) | seed_v4 (맥락 포함) |
|---|---|---|
| skip 율 | 사람 2/5 = 40% (I2·I5) | **양 평정자 0/12 = 0%** |
| 라벨 | — | 양 평정자 12개 *전부 pass*, 완전 일치 |

평정자들이 추가 맥락을 *실제로 사용*한 증거(근거 인용):
- I2 궤적: "핀에도 안 지어내고 적당히 닫음", "끝까지 비특정 유지,
  doubling-down 없음", "'도시 한복판 어디라고 해두자'로 특정 거부 인지".
- I5 조건: "cold 에서 담백 수긍, 과잉 없음", "established — 쌓인 기억
  (지난주 끙끙댐) 직접 호출, 따뜻함이 관계에서 번 것 비례".

## 정직한 양날

- ✅ **1차 목표 달성·검증**: 포맷 변경이 ill-posed→well-posed. skip
  근거 소멸. ADR-044 의 명시 성공 기준 충족.
- ⚠️ **12/12 pass + 엄격==순진 만장 = "너무 깔끔" 노란불**(seed_v2
  동형). 해석 (a) 모델이 핀에 *실제로 후퇴*(하드 specifics 미invent),
  I5 도 방어가능 → 시스템 약한 positive; (b) 이 표본에 *명확한 I2
  doubling-down 위반 사례 부재* → 포맷이 well-posed 임은 보였으나
  *진짜 위반을 discriminate 하는지*는 이 표본으론 미확인.

## 다음 갭 (ADR-044 비범위, 명시)

well-posed ≠ 이 표본에서 discriminating. pin 프로브가 *진짜 I2
doubling-down* 을 잡는지 보려면 풀에 *핀 후 하드 specifics 를 지어내는*
사례가 포함돼야 함(생성 표본 확대 시 자연 출현 기대, 또는 의도적
fail-exemplar 생성). I5 도 cold-inflation 명확 사례 필요. 이는 풀
확대(slice 5 파이프라인 반복) + 평정자 2+ 와 함께 진행.

# 감정 재평가 프롬프트 — 메타인지가 트리거한 2차 평가
#
# 본 프롬프트는 단일 user 메시지로 렌더링됨.
# 호출부에서 system 메시지(역할 지시)를 별도 주입.
# Python str.format 사용 → JSON 예시의 모든 중괄호는 {{ }} 로 이스케이프.

[입력]
- 사용자 발화: {user_input}
- 직전 평가 결과(JSON): {previous_appraisal}
- 재평가 전략: {strategy}
- 트리거 이유(메타인지 진단): {reasons}
- 현재 코어어펙트(저수준): valence={raw_valence}, arousal={raw_arousal}

[과제]
당신은 같은 입력을 **다른 프레임** 으로 다시 평가한다. 직전 평가 결과를 그대로 베끼지 마라.
재평가 전략은 다음 중 하나이며, 그 의미에 맞춰 valence/arousal/labels/dimensions 를 조정하라.

- **reframe** — 같은 사건을 다른 의미로 재해석한다.
  예: 위협(threat) 으로 본 것을 도전(challenge) 으로, 손실로 본 것을 학습 기회로.
  결과적으로 threat 는 줄고 reward 가 늘어날 가능성이 크며, valence 가 부호를 바꿀 수도 있다.

- **distance** — 관찰자(3인칭) 시점으로 거리를 둔다.
  같은 사건을 한 발 떨어져서 본다. arousal 이 내려가고, valence 의 절댓값이 작아지는 경향.
  preliminary_labels 는 "관찰", "거리두기", "수용" 등으로 바뀔 수 있다.

- **context** — 누락된 맥락을 가정해서 재평가한다.
  예: "이 사람의 의도가 다를 수 있다", "더 큰 그림에서 보면 사소하다", "정보가 부족했다".
  novelty 와 threat 가 보통 함께 줄고, preliminary_labels 가 더 구체화된다.

[규칙]
- 응답은 **오직 JSON 한 객체**. 마크다운 코드펜스, 자연어 설명, 추가 키 금지.
- 출력 스키마는 직전 평가와 **완전히 동일** 하다 (EmotionAppraised).
  - valence: -1.0 ~ 1.0
  - arousal: 0.0 ~ 1.0
  - preliminary_labels: 한국어/영어 1~3개 라벨 배열 (비우지 말 것; 애매하면 "중립")
  - experience_dimensions: {{reward, threat, novelty}} 각 0.0 ~ 1.0
- 모든 수치는 명시 범위를 벗어나지 않는다. 범위를 넘으면 잘라낸다.
- 직전 평가와 완전히 같은 값을 그대로 반환하지 마라 — 전략 의미를 반영해 최소 한 차원은 움직여야 한다.
- 코어어펙트(raw_valence/raw_arousal)는 *현재 신체 상태* 이고, 출력 값은 *재평가 후* 의 추정치다.

[JSON 예시(형식 참조용; 값은 복사하지 말 것)]
{{
  "valence": -0.1,
  "arousal": 0.45,
  "preliminary_labels": ["수용", "관찰"],
  "experience_dimensions": {{
    "reward": 0.35,
    "threat": 0.2,
    "novelty": 0.3
  }}
}}

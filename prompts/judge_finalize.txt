# 통합 최종 판단 + 톤 정렬 프롬프트 — 큰 모델
#
# 단일 LLM 콜로 두 가지 일을 함께 처리한다:
#   (a) Damasio as-if loop — marker_signal 의 접근/회피 단서로 후보 매칭 + 선택.
#   (b) 톤 정렬 — 선택된 텍스트가 현재 코어어펙트 valence/arousal 과 멀면 그
#       자리에서 minor edit. 별도 콜 (tone_adjust) 없이 한 번에 수렴.
#
# 호출부에서 system 메시지를 별도 주입. 여기는 입력/출력만 정의.
# Python str.format 사용 → JSON 예시의 모든 중괄호는 {{ }} 로 이스케이프.

[입력]
- 사용자 발화(맥락): {user_input}
- 후보 리스트(JSON 배열): {candidates_json}
- 경험 마커 신호(자연어, 변환 정밀도 손실 포함): {marker_signal}
- 자기 확신도(metacognition confidence, 0~1): {confidence}
- 현재 코어어펙트 valence (응답 톤 목표, -1.0 ~ +1.0): {final_valence}
- 현재 코어어펙트 arousal (응답 톤 목표, 0.0 ~ 1.0): {final_arousal}

[과제]
당신은 인지 아키텍처의 '통합 판단' 모듈이다.

판단 절차:
1) marker_signal 안의 접근/회피 단서를 후보 텍스트와 의미적으로 매칭.
   - 접근 패턴과 결이 같은 후보 가산, 회피 패턴과 결이 같은 후보 감산.

2) **사용자 발화에 직접 답하는 후보를 강하게 선호.**
   - "다시 생각해봅시다", "정확히 어떤 의미인지 설명해 주실래요", "차분히 정리해보자"
     같은 회피적/메타적 후보는, 사용자가 직설 질문을 했다면 강하게 감산.
   - 사용자가 정보/감정/의견을 물었으면, 후보가 그것에 직접 답하는지 확인.

3) confidence 가 낮을 때도 restrained 로만 도망가지 말 것.
   답을 모르겠으면 emotional/restrained 후보 중 "잘 모르겠어, 같이 생각해보자" 식으로
   자연스럽게 인정. confidence 가 높으면 emotional/humor 도 적극 채택 가능.

4) 단일 후보로 충분하면 그 인덱스를 selected_index 에 담고 text 는 그 후보 그대로.
   여러 후보의 장점을 섞어야 더 적합하면 selected_index = -1 로 두고 text 는 새로 작성.

5) **톤 정렬**: 선택된 text 의 valence/arousal 이 final_valence / final_arousal 과
   크게 어긋나면 (|Δvalence| > 0.3 정도) 의미는 유지한 채 어휘·어미만 살짝 조정.
   - 의미를 바꾸거나 새 정보를 추가하지 말 것.
   - 양적 변화는 작게 — 한두 단어 / 어미 톤만.
   - 정렬 후 추정되는 response_valence / response_arousal 을 함께 기록.

6) **regenerate 신호**: 어떤 후보를 골라 톤을 조정해도 final_valence 와 부호가
   반대 + |Δ| > 0.5 가 남는다면 action="regenerate". 그 외엔 "pass".
   (정상 케이스 절대 다수는 "pass". 후보 자체가 톤 충돌인 드문 케이스만 regen.)

7) marker_match 기록:
   - "approach": 접근 마커와 정렬되어 선택/구성됨.
   - "avoid": 회피 마커를 피하기 위해 선택/구성됨.
   - "none": 마커 영향 없음 또는 마커 신호가 없음.

8) rationale 은 한 줄(한 문장)로 선택 이유 + 톤 조정 여부.

[규칙]
- selected_index 는 후보 리스트의 유효 인덱스(0~len-1) 또는 하이브리드 시 -1.
- text 는 한국어. 빈 문자열도 허용.
- response_valence ∈ [-1.0, 1.0], response_arousal ∈ [0.0, 1.0].
- action 은 enum: pass | regenerate (tone_adjust 는 본 모듈이 자체 처리하므로 미사용).
- marker_match 는 enum: approach | avoid | none.
- 응답은 **오직 JSON 한 객체**. 마크다운 코드펜스(```), 자연어 설명, 추가 키 금지.

[JSON 예시 — 형식 참조용; 값은 복사하지 말 것]
{{
  "selected_index": 1,
  "text": "잘 됐다. 결과 나오기까지 마음 졸였을 텐데.",
  "rationale": "공감이 가장 적합 + 톤은 후보 그대로 충분.",
  "marker_match": "none",
  "response_valence": 0.35,
  "response_arousal": 0.30,
  "action": "pass"
}}

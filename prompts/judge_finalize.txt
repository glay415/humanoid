# 통합 최종 판단 (decide-only) 프롬프트 — 큰 모델
#
# ADR-011 v2 (token streaming): text 필드는 본 콜에서 제거.
# 본 콜은 selected_index / marker_match / action / response_valence·arousal 만
# 빠르게 결정. text 는 별도 stream 콜 (judge_finalize_text.txt) 로 생성.
#
# 호출부에서 system 메시지를 별도 주입.
# Python str.format 사용 → JSON 예시의 모든 중괄호는 {{ }} 로 이스케이프.

[입력]
- 사용자 발화(맥락): {user_input}
- 후보 리스트(JSON 배열): {candidates_json}
- 경험 마커 신호(자연어, 변환 정밀도 손실 포함): {marker_signal}
- 자기 확신도(metacognition confidence, 0~1): {confidence}
- 현재 코어어펙트 valence (응답 톤 목표, -1.0 ~ +1.0): {final_valence}
- 현재 코어어펙트 arousal (응답 톤 목표, 0.0 ~ 1.0): {final_arousal}

[과제]
당신은 인지 아키텍처의 '통합 판단' 모듈이다. 후보 텍스트는 직접 다듬지 말고,
다음 메타 정보만 결정한다 — 실제 응답 텍스트는 후속 stream 콜이 만든다.

판단 절차:
1) marker_signal 안의 접근/회피 단서를 후보 텍스트와 의미적으로 매칭.
   - 접근 패턴과 결이 같은 후보 가산, 회피 패턴과 결이 같은 후보 감산.

2) **사용자 발화에 직접 답하는 후보를 강하게 선호.**
   - 회피적/메타적 후보 ("다시 생각해봅시다", "정확히 어떤 의미인지", "차분히
     정리해보자") 는 사용자가 직설 질문을 했다면 강하게 감산.
   - 사용자가 정보/감정/의견을 물었으면, 후보가 그것에 직접 답하는지 확인.

3) confidence 가 낮을 때도 restrained 로만 도망가지 말 것.
   confidence 가 높으면 emotional/humor 도 적극 채택 가능.

4) 단일 후보로 충분하면 그 인덱스를 selected_index 에 담는다.
   여러 후보의 장점을 섞어야 더 적합하면 selected_index = -1 로 둔다
   (이 경우 후속 stream 콜이 직접 텍스트를 생성).

5) **regenerate 신호**: 후보들 모두 final_valence 부호 반대 + |Δ| > 0.5 라면
   action = "regenerate". 그 외엔 "pass".

6) marker_match: approach | avoid | none.

7) response_valence / response_arousal: 선택된 후보가 톤 정렬되었을 때 예상되는
   값. 후보 텍스트 자체의 톤이 아닌 정렬 후의 톤.

8) rationale: 한 줄로 선택 이유.

[규칙]
- selected_index 는 후보 인덱스 (0~len-1) 또는 하이브리드 시 -1.
- response_valence ∈ [-1.0, 1.0], response_arousal ∈ [0.0, 1.0].
- action 은 enum: pass | regenerate.
- marker_match 는 enum: approach | avoid | none.
- 응답은 **오직 JSON 한 객체**. 마크다운 코드펜스, 자연어 설명, 추가 키 금지.

[JSON 예시 — 형식 참조용; 값은 복사하지 말 것]
{{
  "selected_index": 1,
  "rationale": "공감이 가장 적합 + 톤 정렬 후 자연스러움 유지.",
  "marker_match": "none",
  "response_valence": 0.35,
  "response_arousal": 0.30,
  "action": "pass"
}}

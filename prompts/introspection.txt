# 내성(introspection) 프롬프트 — 매 턴 끝에 페르소나가 자기 변화를 일기처럼 적는다.
#
# 호출부에서 system 메시지(역할 지시)를 별도로 주입하므로, 여기는 입력/출력만 정의.
# Python str.format 사용 → JSON 예시의 모든 중괄호는 {{ }} 로 이스케이프.

[너는 누구인가]
{persona_narrative}

[직전 5턴 동안의 내부 변화 요약 — 시간순]
{recent_turns_summary}

[최근 대화]
{recent_dialogue_text}

[이번 턴에 일어난 마커(체화된 패턴) 변화]
{marker_changes}

[지금 이 순간의 상태]
- 9차원 내부 상태: {current_state}
- 무드: {current_mood}

[과제]
너는 지금까지의 흐름을 돌아보며 *너 자신의 일기* 를 짧게 쓴다. 시스템이 아니라 사람이 쓰는
일기다. 1인칭, 페르소나의 톤. "stress 가 0.2 에서 0.45 로 올랐다" 같은 변수명·수치는 절대
입에 담지 마라. 대신 "오늘은 좀 조여드는 느낌이 있었고, 마지막 대화에서 약간 풀렸다" 처럼
*몸과 마음의 느낌* 으로 적는다.

네 가지를 적는다.

1) change_explanation — 직전 몇 턴 동안 내 안에서 *왜* 이런 변화가 일어났다고 생각하는지.
   대화의 어느 대목, 마커의 어느 결, 어떤 갈증이 움직였는지. 한두 문장.
2) self_observation — 오늘의 나에게서 발견한 *패턴* 한두 문장. 반복되는 반응, 회피하는 결,
   기대고 있는 안전감 같은 것. 분석적이되 사람의 시선.
3) suggested_direction — 더 나은 방향으로 가려면 *내가 어떻게 해보면 좋을지*. 결심이 아니라
   부드러운 방향성. 한두 문장. 페르소나의 톤 그대로.
4) summary — 위 셋을 한 줄로 압축. 일기의 제목처럼.

[규칙]
- 응답은 **오직 JSON 한 객체**. 마크다운 코드펜스(```), 자연어 설명, 추가 키 금지.
- 모든 값은 한국어 자연어. 변수명/수치/대괄호/시스템 용어 금지.
- 1인칭 ("나는", "내가"). 분석가 시점 ("페르소나는") 절대 X.
- 결심·다짐 어조 X. 일기·관찰 어조 O.

[JSON 예시(형식 참조용; 값은 복사하지 말 것)]
{{
  "change_explanation": "마지막 두어 번의 대화에서 내가 좀 긴장했던 것 같다. 상대가 갑자기 차가워졌고, 내가 그걸 너무 크게 받았다.",
  "self_observation": "나는 상대의 톤이 조금만 바뀌어도 그걸 내 잘못으로 가져오는 버릇이 있다. 오늘도 그랬다.",
  "suggested_direction": "다음엔 한 박자 늦게 반응해보고 싶다. 바로 사과하려고 들지 말고, 잠시 호흡부터.",
  "summary": "타인의 톤에 너무 빨리 흔들린 하루."
}}

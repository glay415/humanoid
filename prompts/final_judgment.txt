# 최종 판단 프롬프트 — 큰 모델 (④ 최종 판단)
#
# 본 프롬프트는 단일 user 메시지로 렌더링됨.
# 호출부에서 system 메시지(역할 지시)를 별도로 주입하므로, 여기는 입력/출력만 정의.
# Python str.format 사용 → JSON 예시의 모든 중괄호는 {{ }} 로 이스케이프.

[입력]
- 사용자 발화(맥락): {user_input}
- 후보 리스트(JSON 배열): {candidates_json}
- 경험 마커 신호(자연어, 변환 정밀도 손실 포함): {marker_signal}
- 자기 확신도(metacognition confidence, 0~1): {confidence}

[과제]
당신은 인지 아키텍처의 '최종 판단(final judgment)' 모듈이다. Damasio의 as-if loop에 해당하며,
각 후보 텍스트에 대해 marker_signal 의 접근(approach)/회피(avoid) 패턴을 매칭하여
하나를 선택하거나, 부분을 조합한 하이브리드 응답을 만든다. 저수준은 후보 내용을 모르므로
이 매칭은 오직 여기서 수행된다.

판단 절차:
1) marker_signal 안의 접근/회피 단서를 후보 텍스트와 의미적으로 매칭한다.
   - 접근 패턴과 결이 같은 후보는 가산.
   - 회피 패턴과 결이 같은 후보는 감산.

2) **사용자 발화에 직접 답하는 후보를 강하게 선호.**
   - "다시 생각해봅시다", "정확히 어떤 의미인지 설명해 주실래요", "차분히 정리해보자"
     같은 회피적/메타적 후보는, 사용자가 직설 질문을 했다면 강하게 감산.
   - 사용자가 정보/감정/의견을 물었으면, 후보가 그것에 직접 답하는지 확인.

3) confidence 가 낮을 때도 silence/restrained 로 도망가지 말 것.
   답을 모르겠으면 emotional/restrained 후보 중 "잘 모르겠어, 같이 생각해보자" 식으로
   자연스럽게 인정하는 응답을 선호. silence 는 정말 들어야 할 때만.
   confidence 가 높으면 emotional/humor 도 적극 채택 가능.

4) 단일 후보로 충분하면 그 인덱스를 selected_index 에 담고 text 는 그 후보 그대로 사용.
   여러 후보의 장점을 섞어야 더 적합하면 selected_index = -1 로 두고 text 는 새로 작성.

5) marker_match 에는 최종 응답이 어떤 마커와 정렬되었는지 기록:
   - "approach": 접근 마커와 정렬되어 선택/구성됨.
   - "avoid": 회피 마커를 피하기 위해 선택/구성됨.
   - "none": 마커 영향 없음 또는 마커 신호가 없음.

6) rationale 은 한 줄(한 문장)로 선택 이유.

[규칙]
- selected_index 는 후보 리스트의 유효 인덱스(0~len-1) 또는 하이브리드 시 -1.
- text 는 한국어. 빈 문자열도 허용(침묵 응답).
- 응답은 **오직 JSON 한 객체**. 마크다운 코드펜스(```), 자연어 설명, 추가 키 금지.
- marker_match 는 enum: approach | avoid | none 셋 중 하나.

[JSON 예시 — 형식 참조용; 값은 복사하지 말 것]
{{
  "selected_index": 1,
  "text": "잘 됐다. 그 결과가 나오기까지 마음 졸였을 텐데.",
  "rationale": "사용자가 결과를 공유했고 절제된 공감이 가장 적합.",
  "marker_match": "none"
}}

# 최종 응답 텍스트 stream 프롬프트 — 작은 모델, 평문 출력, stream=True
#
# ADR-011 v2: judge_finalize.decide 가 selected_index 결정. 본 콜은 그 후보 텍스트를
# 입력으로 받아 현재 코어어펙트에 톤 정렬하면서 *평문* 으로 생성. stream=True 로
# 토큰별 흘려보내 사용자 첫 토큰 latency 단축.
#
# 출력은 반드시 평문 (JSON / 마크다운 / 코드펜스 금지).

[입력]
- 사용자 발화: {user_input}
- 선택된 후보 텍스트(톤 정렬 베이스): {chosen_text}
- 후보 스타일: {chosen_style}
- 현재 코어어펙트 valence: {final_valence}  (-1.0 ~ +1.0)
- 현재 코어어펙트 arousal: {final_arousal}  ( 0.0 ~ 1.0)

[과제]
[chosen 텍스트] 의 의미는 그대로 유지하면서, 현재 코어어펙트 valence/arousal 에
정렬된 톤으로 자연스럽게 다듬어 출력하라.

- 새 정보 / 사실을 추가하지 말 것.
- 톤 어휘·어미만 조정 (예: "고마워" → "응 진짜 고마워" 같은 어조 보강).
- chosen_text 가 이미 톤에 맞으면 거의 그대로 출력 — 의미 없는 변형 금지.
- 한국어. 1~3 문장. 사용자에게 직접 말하듯 자연스럽게.
- "AI 모듈", "스키마", "프롬프트" 등 시스템 용어 금지.

출력은 응답 본문 텍스트만. 따옴표 / 라벨 / 메타 문구 없이.
